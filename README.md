# HamAlert Bot 📡

Connects to **hamalert.org:7300** via Telnet, parses incoming JSON spots,
and forwards them via Gmail.

---

## ⚠️ Security & Privacy — Read Before Pushing to GitHub

This project is designed to be **safe to publish publicly**:

| What | How it's protected |
|---|---|
| Credentials (passwords, API keys) | Stored only in `.env` — **git-ignored**, never committed |
| Email addresses | Stored only in `.env` — **git-ignored**, never committed |
| Callsign / username | Stored only in `.env` — **git-ignored**, never committed |
| Safe template | `.env.example` contains **no real values** — safe to commit |

**Never** put real credentials in `docker-compose.yml`, `hamalert.py`, or any other tracked file.

If you accidentally committed a secret:
1. Rotate/revoke it immediately (change HamAlert password, revoke Gmail App Password)
2. Use `git filter-repo` or [BFG Repo Cleaner](https://rtyley.github.io/bfg-repo-cleaner/) to scrub history
3. Force-push the cleaned history

## Setup

### 1 · Clone and create your secrets file

```bash
git clone https://github.com/YOUR_USER/hamalert-bot.git
cd hamalert-bot
cp .env.example .env          # .env is git-ignored
```

### 2 · Create a Gmail App Password

Gmail requires an **App Password** — not your regular password — for SMTP:

1. Go to your Google Account → **Security**
2. Enable **2-Step Verification** (required)
3. Visit https://myaccount.google.com/apppasswords
4. Create a new entry → **Mail / Other (custom name)**
5. Copy the generated **16-character password**

> Use a **dedicated Gmail account** for the bot, not your personal one.
> That way you can revoke access independently if needed.

### 3 · Fill in `.env`

```dotenv
HA_USERNAME=CALLSING
HA_PASSWORD=your_hamalert_password

GMAIL_USER=your-bot@gmail.com
GMAIL_APP_PW=abcd efgh ijkl mnop

EMAIL_TO=target_email@gmail.com
```

### 4 · Build & run

```bash
docker compose build
docker compose up -d
docker compose logs -f   # watch live
```

## Environment Variables Reference

| Variable | Required | Description |
|---|---|---|
| HA_USERNAME | yes | HamAlert login (callsign) |
| HA_PASSWORD | yes | HamAlert password |
| GMAIL_USER | yes | Gmail sender address |
| GMAIL_APP_PW | yes | Gmail App Password (16 chars) |
| EMAIL_TO | yes | Recipient e-mail address |
| BATCH_SIZE | no | Spots per email (1 = instant, default 1) |
| RECONNECT_DELAY | no | Seconds before reconnect on drop (default 30) |

## Troubleshooting

| Symptom | Fix |
|---|---|
| Missing required environment variables | Ensure .env exists and all five required vars are set |
| 535 Authentication failed | Use an App Password, not your Gmail password |
| Connection refused | Port 7300 must be open for outbound TCP |
| Non-JSON lines in logs | Normal — HamAlert sends text pings and banners |
