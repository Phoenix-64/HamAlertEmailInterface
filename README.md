# HamAlert Bot 📡

Connects to **hamalert.org:7300** via Telnet, parses incoming JSON spots,
and forwards them via Gmail. Designed to run as a Portainer stack — no local
setup required beyond pasting one file.

---

## Repository structure

```
hamalert-bot/
├── hamalert.py          # bot logic
├── Dockerfile           # image definition
├── docker-compose.yml   # Portainer stack definition (no secrets)
├── .gitignore
└── README.md
```

No `.env` file is used. All secrets are entered directly in the Portainer UI
and never touch the filesystem or the repository.

## Deployment via Portainer

### 1 · Prerequisites

| Requirement | Notes |
|---|---|
| Portainer | With access to a Docker host |
| HamAlert account | Free at [hamalert.org](https://hamalert.org) |
| Gmail App Password | See step 2 below |

### 2 · Create a Gmail App Password

Gmail requires an **App Password** — not your regular password — for SMTP:

1. Go to your Google Account → **Security**
2. Enable **2-Step Verification** (required)
3. Visit https://myaccount.google.com/apppasswords
4. Create a new entry → **Mail / Other (custom name)**
5. Copy the generated **16-character password**

Use a **dedicated Gmail account** for the bot, not your personal one,
so you can revoke access independently if needed.

### 3 · Deploy

1. In Portainer: **Stacks → Add Stack**
2. Give the stack a name (e.g. `hamalert-bot`)
3. Paste the contents of `docker-compose.yml` into the editor
4. Scroll down to **Environment variables** and add:

| Variable | Description |
|---|---|
| `HA_USERNAME` | HamAlert callsign / login |
| `HA_PASSWORD` | HamAlert password |
| `GMAIL_USER` | Gmail sender address |
| `GMAIL_APP_PW` | Gmail App Password (16 chars) |
| `EMAIL_TO` | Recipient e-mail address |

5. Click **Deploy the stack**

Portainer clones the GitHub repo, builds the image from the `Dockerfile`,
injects the environment variables, and starts the container.

### 4 · Optional tuning

Set these in the Portainer environment variables section as needed:

| Variable | Default | Description |
|---|---|---|
| `BATCH_SIZE` | `1` | Spots per email (`1` = send instantly) |
| `RECONNECT_DELAY` | `30` | Seconds before reconnect after a dropped connection |

## Security

- No secrets are stored in this repository
- All credentials are entered in the Portainer UI and injected at runtime
- The compose file and all code are safe to keep in a public repo

If you ever accidentally commit a secret:
1. **Immediately** rotate it (change the HamAlert password / revoke the Gmail App Password)
2. Scrub the git history with [BFG Repo Cleaner](https://rtyley.github.io/bfg-repo-cleaner/) or `git filter-repo`
3. Force-push the cleaned history

## Troubleshooting

| Symptom | Fix |
|---|---|
| `Missing required environment variables` | Check all five vars are set in the Portainer stack editor |
| `535 Authentication failed` | Use an App Password, not your Gmail password |
| `Connection refused` | Port 7300 must be open for outbound TCP on the Docker host |
| Non-JSON lines in logs | Normal — HamAlert sends text pings and banners |
